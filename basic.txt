
    In practice, the use of typical Gaussian elimination is the backbone for many physical and scientific applications that
involve the solving of linear systems with respect to the form Ax = b. This algorithm in theory would solve in O(N^3), where 
looping through the rows and columns, plus the backsubstitution would create a triple nested for-loop. However in the case of 
using a symmetric positive definite matrix (SPD), we can further break and decompose this process down into segmented parts for 
solving using the LUP algorithm which would result in a speed optimization.

    Analyzing the graphs and results from timing, the LUP-algorithm follows a similar but differing growth pattern to O(N^3), where
sample sizes of N = 10 is practically identical, however the interest lies in sample size N = 100. In this test case, we can see that
the slope of work done to solve this matrix is more akin to the complexity of O(N^2). I believe this is due to the fact that since N is moderately sized, the algorithm's solving is dominated by the 
backward and forward substitution methods which solve in O(N^2), where the decomposition is offloaded to a complexity similar to O(1/3*N^3). This explanation
would make the most sense, as looking at the case when N = 1000, our slope rises to approximately O(N^3.5), which I belive is due to memory access
and latency issues more than the actual algorithm itself. Regardless, this case study proves with proper scaling, LUP decomposition can properly be 
deployed for efficient solving of systems Ax = b
